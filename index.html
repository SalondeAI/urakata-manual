全体概要

オフ会の目的

ウラカタマッチングサロンのオフ会は、コンテンツホルダー（経営者）とウラカタ（裏方人材）のリアルマッチングの場です。オンラインでは生まれにくい深い繋がりを作り、実際のビジネスマッチングにつなげることを目指しています。

  • 作業会（交流会）：アイスブレイク→自由交流→スペシャルコンテンツの構成
  • 懇親会：飲食を伴うカジュアルな交流の場
  • 二次会：任意参加の延長交流（会場別手配）

名札紐の色で参加者の属性がひと目でわかる仕組み。交流時のマッチング効率を高めます。

顔出しNGの方への配慮

  • 黄色いハチマキを着用 → 写真・動画への映り込みNG
  • 写真アップロード時は黄色ハチマキの方の顔を隠す
  • 受付時にハチマキを配布し、注意事項を口頭でも伝達

運営の基本原則

  1. 安全第一：参加者の安全と快適さが最優先
  2. 時間厳守：会場の撤収時間は絶対。逆算してスケジュール管理
  3. 声かけ文化：孤立者がいたら積極的に声をかける
  4. 柔軟対応：想定外のトラブルはチームで相談して解決
  5. 振り返り重視：毎回の学びを次回に活かす

タイムテーブル運用のポイント

  • 席替え：30分ごと。簡単なルール（例：「右隣のテーブルへ」）で素早く
  • 撤収時間：会場契約時間の15分前には原状復帰完了を目標
  • バッファ：予定より5〜10分早めに次の工程準備を開始
  • 懇親会入場：入口で滞留しやすいため、アナウンスで分散入場を促す

司会進行チーム

体制

  • MC（司会）：1名 — 進行の主導
  • 司会補佐（タイムキーパー）：1名 — スライド操作・時間管理

※ 司会と受付の兼任は避ける（過去の教訓：両立困難で抜け漏れ発生）

事前準備タスク

  1. スライド投映チェック（PC接続・HDMI/Type-C変換確認）
  2. マイク・スピーカーの動作テスト
  3. Wi-Fi情報の確認（会場から取得してスライドに反映）
  4. BGM準備（Spotify等、著作権フリーの音源）
  5. 主宰・登壇者との進行すり合わせ
  6. アイスブレイクのテーマ決定（2〜3個）
  7. カウントダウンタイマーの準備
  8. アフターアンケートURL/QRコードの確認

過去の好事例

  • スライド・マイクの事前チェックを開場30分前に完了 → トラブルなし
  • 質問を投げかけ形式でアイスブレイク → 盛り上がりやすい

  • カウントダウンタイマー表示 → 時間管理が可視化されて好評

  • 司会と受付兼任で抜け漏れ → 兼任禁止
  • 建物入口で混雑 → エントランス係を配置し案内板掲示

受付チーム

体制（100名超の場合）

役割人数詳細
受付リーダー1名全体指揮、トラブル対応判断
名簿チェック4名4窓口体制（あ行〜/か行〜/さ行〜/た行〜 等で分担）
名札・配布物2名名札ホルダー渡し、リストバンド配布
事務/当日対応1名当日申込（QRコード決済）、キャンセル処理

受付の流れ

  1. 参加者が到着 → 名前を確認
  2. スプレッドシート（タブレット/PC）で名簿チェック → チェックマーク
  3. 名札ホルダー（紐の色で属性別）を渡す
  4. 顔出しNGの方には黄色ハチマキを渡す
  5. リストバンド配布（作業会/懇親会の区別用）
  6. 会場内への案内

名簿管理

  • Googleスプレッドシートで管理（紙は不要）
  • 事前にあいうえお順でソート → 検索効率UP
  • 4窓口に分割（頭文字で振り分け）→ 列を短くして待ち時間削減
  • 当日キャンセル・追加参加の列を別途用意
  • UTAGE等の決済システムと名簿の照合を事前に済ませておく

名札の準備

  • 名札ホルダー：紐の色で属性識別（赤/青/黄）
  • 名前は両面印刷（裏返しても見える）
  • ジャンル・得意分野も名札に記載 → マッチング促進
  • 顔出し有無も名札に小さく記載（過去改善）
  • あいうえお順に仕分け → 受付時にすぐ見つけられる
  • 空白の名札を数枚用意 → 当日申込・名前不一致への対応
  • 名札紐の長さ調整：長すぎると不評（過去フィードバック）

受付テーブル配置

  • 会場内に設置（エレベーターホール前は狭くて滞留するため）
  • 4窓口（100名超の場合）を横並びに配置
  • 動線：入口 → 受付テーブル → 名札受取 → 座席へ
  • テーブル前に「あ行〜」等の案内表示を掲示

受付ローテーション

受付は立ちっぱなしで負荷が高いため、3部制ローテーションを推奨：

  1. 第1部（13:30〜14:30）：ピークタイム。全員配置
  2. 第2部（14:30〜16:00）：半数が交代で休憩・交流参加
  3. 第3部（16:00〜17:00）：最低2名で遅刻者・再入場対応

過去の好事例

  • あいうえお順ソート → 検索時間が大幅に短縮
  • ストラップ色分け → 属性がひと目でわかり好評
  • 懇親会ローテーション → スタッフも楽しめた
  • カラー印字の名札 → 視認性が良く好評
  • 名札両面印刷 → 裏返しても名前が見えて便利

過去の失敗・注意点

  • 名札ホルダーの紐が絡まる → 事前にほどいて準備
  • 名札ホルダー不足 → 予備を2割多めに用意
  • UTAGE未反映の参加者 → 事前に決済リストと名簿を照合
  • 当日申込フローが決まっていない → QRコード決済のフローを事前に確定
  • 受付での名前不一致 → Discord名・本名の対応表を事前作成

タスク詳細

レイアウトのポイント

  • 島型配置：6〜8人掛け×必要数。席替えしやすい
  • 通路幅：人がすれ違える幅を確保
  • ゲスト用スペース：個別コンサル用の列整理スペースを確保
  • まぎーさん囲み会：椅子10脚を輪にして配置（テーブルなし）
  • レイアウト図を事前に作成・共有しておくとスムーズ

過去の好事例

  • レイアウト図を紙面で全スタッフに共有 → 迷わず配置
  • 声掛け合いながらのサポート → チームワーク向上
  • 原状復帰の素早さ → 会場側から好評

注意点

  • 受付チームとの認識共有（誰がどこまで対応するか）を事前に決める
  • 会場内誘導のローテーションを決めておく（ずっと同じ人が立ちっぱなしにならないよう）
  • 車椅子利用者がいる場合の動線確保

エントランスチーム

役割と配置

100名超の大規模イベントでは、ビル入口〜会場フロアまでの案内係が必須です。

  • 配置場所：ビル1Fエントランス、エレベーターホール前
  • 人数：2名（1F + 会場フロア）
  • 役割：
    • 参加者に声をかけて会場フロアへ案内
    • エレベーターの案内（複数基ある場合どれに乗るか）
    • 混雑時の滞留防止（エレベーターホールが狭い場合は特に重要）

事前準備

  • 下見：可能であれば事前にビル入口〜会場までのルートを確認
  • 案内資料：駅から会場までのルート写真・動画を作成してDiscordで共有
  • 案内板：ビル入口に「ウラカタマッチングサロン オフ会 ○F」の掲示

  • Googleフォトの共有アルバムを事前に作成
  • 集合写真は当日中にDiscordで共有
  • 懇親会でのインタビューはお酒後が自然体で良い内容が撮れる（過去実績）

  • 動画撮影機材 + 三脚/ジンバル
  • SDカード予備
  • インタビュー質問リスト（固定テンプレ推奨）
  • 写真格納用Googleフォトフォルダ

ゲストアテンドチーム

担当範囲

  • ゲストの送迎（駅〜会場）
  • 会場内での案内・席への誘導
  • ゲスト用の飲み物・軽食の準備
  • 個別コンサル時の列整理
  • ゲストのスケジュール管理（退出時間等）

事前準備

  • 会場付近のルートを事前に下見（いづまるさん方式）
  • ゲスト用の飲み物は前日に購入し、指定座席に配置
  • ゲストの到着時間・退出時間を事前確認
  • ゲストの好み（飲み物等）を事前にリサーチ

個別コンサル運営（おさるさん方式）

  • 並び順で時間の限り対応（予約制ではない）
  • 会場係の1名が列整理を担当
  • 1人あたりの目安時間を設定（5〜10分）
  • 列が長い場合は途中で締め切りアナウンス
